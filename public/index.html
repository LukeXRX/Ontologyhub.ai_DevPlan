🔴 높음 1. API 인증 없음

백엔드 API에 인증이 없어서 누구나 엔드포인트 호출 가능

해결: FastAPI에 JWT Bearer 토큰 미들웨어 추가. NextAuth → JWT 발급 → 모든 API에 Authorization: Bearer {token} 강제

from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
import jwt

security = HTTPBearer()

async def verify_token(credentials: HTTPAuthorizationCredentials = Depends(security)):
    payload = jwt.decode(credentials.credentials, SECRET_KEY, algorithms=["HS256"])
    return payload["user_id"]

🔴 높음 2. Admin 인증 — sessionStorage

sessionStorage에 비밀번호 저장, 클라이언트에서만 체크

해결: NextAuth.js 서버사이드 세션으로 교체 + httpOnly 쿠키 + 미들웨어 차단

// middleware.ts
export { default } from "next-auth/middleware"
export const config = { matcher: ["/dashboard/:path*", "/nebula/:path*"] }

🔴 높음 3. Cypher 인젝션 위험

/api/graph/query에서 사용자 입력이 Cypher 쿼리에 직접 삽입

해결: 엔드포인트 삭제 → 파라미터화된 전용 엔드포인트로 대체

result = neo4j.run(
    "MATCH (n) WHERE elementId(n) = $id AND n.owner_id = $uid RETURN n",
    {"id": node_id, "uid": user_id}
)

🟡 중간 6. Google OAuth 테스트 모드

해결: 프로덕션 전환 신청 (개인정보 처리방침 URL + 서비스 약관 + 앱 로고 제출). Incremental Auth 방식 — 기본 로그인 먼저 승인, 데이터 연동은 별도 요청 (CASA Tier 2, 2~4주)

Phase 2

Conversational Ontology Engine

AI 대화형 온톨로지 설계 — 핵심 차별화 기능
기획 의도

OntologyHub의 핵심 차별화 기능. 현재는 Google 데이터를 AI가 자동 추출하는 수동적 파이프라인. Phase 2에서는 사용자가 AI와 '대화'하면서 자신의 온톨로지를 설계하는 능동적 경험을 구축한다.

"나는 제조업에서 품질관리를 담당하는데, 우리 공장의 공정 온톨로지를 만들고 싶어" → AI가 스키마를 제안 → 사용자가 수정 → 최종 확정

기존 Protégé/TopBraid는 OWL/RDF를 알아야 하지만, OntologyHub는 자연어만 알면 된다.
