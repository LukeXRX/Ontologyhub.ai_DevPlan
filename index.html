01. Protégé 소개

개발 초보도 이해할 수 있는 쉬운 설명

한 마디로: "온톨로지계의 Photoshop"

Protégé(프로테제)는 Stanford 대학에서 만든 무료 온톨로지 편집기입니다. 1999년에 처음 나와서 지금까지 25년 넘게 업데이트되고 있습니다. 전 세계 36만 6천 명 이상의 연구자, 기업이 사용 중입니다.

온톨로지란, "세상의 지식을 컴퓨터가 이해할 수 있게 구조화하는 것"입니다. 예를 들어 "사과는 과일이다", "과일은 음식이다" 같은 관계를 정의하면, 컴퓨터가 "그러면 사과는 음식이다"라고 스스로 추론할 수 있습니다.

비유하면: Figma가 "디자인"을 만드는 툴이라면, Protégé는 "지식 구조"를 만드는 툴입니다. 우리 OntologyHub는 여기에 AI를 더해서 "개인 지식 구조를 자동으로 만들어주는" 툴입니다.

두 가지 버전이 있다

🖥️

Protégé Desktop

컴퓨터에 설치해서 쓰는 버전. 기능이 아주 많고 강력합니다. Java로 만들어져서 Mac/Windows/Linux 전부 됩니다. 추론 엔진(Reasoner)을 직접 연결해서 "컴퓨터가 스스로 새로운 지식을 찾아내는" 것이 가능합니다.

v5.6.8 / Desktop App
🌐

WebProtégé

웹 브라우저에서 쓰는 버전. 설치 없이 바로 사용. 여러 사람이 동시에 편집할 수 있는 "협업 기능"이 핵심입니다. 구글 독스처럼 실시간으로 같이 수정 가능합니다.

v4.2.64 / Web-based

03. 핵심 기능 상세

Protégé의 특별한 기능들을 쉽게 풀어서 설명합니다

추론 엔진
온톨로지 편집
시각화
표준 & 포맷
플러그인

🧠 Reasoner (추론기)

온톨로지에 정의된 규칙을 바탕으로 컴퓨터가 "스스로 새로운 사실을 발견"하는 기능입니다.

쉬운 예시: "고양이는 동물이다", "동물은 생물이다"라고 정의하면, 추론기가 자동으로 "고양이는 생물이다"를 알아냅니다.

🔍 DL Query (논리 질의)

Manchester 문법으로 온톨로지에 질문을 던질 수 있습니다.

쉬운 예시: "프로그래밍 언어이면서 Google이 만든 것"이라고 질의하면, Go, Dart 등이 결과로 나옵니다.

📜 SWRL Rules (규칙 언어)

Semantic Web Rule Language. "만약 A이면 B이다"라는 규칙을 직접 작성할 수 있습니다.

쉬운 예시: "만약 어떤 사람이 부모가 있고, 그 부모에게 형제가 있으면, 그 형제는 이 사람의 삼촌/이모이다"

✅ Consistency Check

온톨로지 안에 모순이 없는지 자동으로 검사합니다.

왜 중요한가: 온톨로지가 커지면 사람이 일일이 모순을 확인할 수 없습니다. Reasoner가 수만 개의 규칙을 논리적으로 검증합니다.

🌳 Class Hierarchy

트리 구조로 개념의 상-하 관계를 정의합니다.

쉬운 예시: "음식 → 과일 → 사과" 이런 식으로 위-아래 관계를 트리로 만듭니다. 무한 깊이 가능.

🔗 Object Property (관계 속성)

"A가 B에게 ~하다"라는 관계를 정의합니다. 역관계, 함수 속성 등 설정 가능.

쉬운 예시: "isParentOf"를 정의하면, 자동으로 역관계인 "isChildOf"도 설정 가능.

📝 Data Property (데이터 속성)

엔티티에 값(숫자, 문자, 날짜 등)을 붙이고 타입 제한도 가능합니다.

쉬운 예시: "hasAge(나이)"를 추가하면, 값이 0~150 사이 정수여야 한다는 제약 가능.

⚖️ Class Restrictions

클래스에 논리적 조건을 붙입니다. "모든(only)", "일부(some)", "정확히 N개(exactly)" 같은 제약.

쉬운 예시: "채식주의자 = 음식을 먹되, only 식물성 음식만 먹는 사람" — 자동으로 육류 섭취자는 제외.

🌐 OWLViz

클래스 계층을 그래프로 보여줍니다.

특별한 점: "추론 전 vs 추론 후"를 나란히 비교해서, AI가 새로 발견한 관계를 시각적으로 확인 가능.

🕸️ OntoGraf

인터랙티브 그래프 탐색 도구. 노드를 클릭하면 연결된 관계가 펼쳐집니다.

NebulaView와 비슷하지만, 관계 유형별 필터링, 계층 깊이 제어 등이 더 세밀합니다.

📐 VOWL (Visual OWL)

OWL 온톨로지를 시각적으로 표현하는 국제 표기법. Force-directed 그래프 레이아웃 사용.

쉬운 예시: 클래스는 큰 원, 프로퍼티는 화살표 — OWL 문법을 몰라도 그림만 보면 이해 가능.

📊 OWL2UML

OWL 온톨로지를 자동으로 UML 클래스 다이어그램으로 변환합니다.

왜 유용한가: 개발자들이 익숙한 UML로 변환하면, 온톨로지를 모르는 개발자도 구조 이해 가능.

🏛️ OWL 2 (Web Ontology Language)

W3C 국제 표준 온톨로지 언어.

우리와의 차이: OntologyHub는 자체 스키마(ALIVE Schema)를 사용. OWL 2로 변환되면 전 세계 도구와 호환 가능.

📄 RDF / Turtle / JSON-LD

온톨로지 데이터를 저장하는 다양한 포맷을 지원합니다.

쉬운 예시: RDF/XML은 기계용, Turtle은 사람이 읽기 쉽게, JSON-LD는 웹 개발자용.

🔎 SPARQL

온톨로지/RDF 데이터를 쿼리하는 표준 질의어. SQL의 온톨로지 버전.

쉬운 예시: "나이가 30 이상이고, 서울에 사는 사람의 이름을 알려줘" 같은 질문을 코드로 표현.

🌍 Ontology Import

다른 온톨로지를 불러와서 내 온톨로지에 합칠 수 있습니다.

쉬운 예시: DBpedia(위키피디아의 온톨로지 버전)를 불러와서 내 온톨로지와 합칠 수 있음.

🔌 플러그인 아키텍처

Protégé의 가장 큰 강점. 누구나 기능을 확장하는 플러그인을 만들 수 있습니다.

쉬운 비유: Chrome 브라우저의 "확장 프로그램"과 같습니다.

📊 Cellfie (스프레드시트→온톨로지)

엑셀/CSV 파일의 데이터를 온톨로지로 자동 변환하는 플러그인입니다.

왜 중요한가: 기업들의 데이터 대부분은 엑셀에 있습니다. 이걸 온톨로지로 바꿀 수 있으면 기존 데이터 활용 가능.

🗄️ DB 연결 플러그인

MySQL, PostgreSQL 등 관계형 DB의 데이터를 온톨로지로 매핑하는 플러그인입니다.

쉬운 예시: 회사의 고객 DB를 온톨로지에 연결하면, 추론으로 복잡한 조건의 고객을 찾을 수 있음.

🔬 SNAP SPARQL Query

Protégé 안에서 직접 SPARQL 쿼리를 실행하고 결과를 보는 플러그인입니다.

쉬운 예시: 터미널 없이 에디터 안에서 바로 질문하고 답을 받을 수 있습니다.

07. Gap 분석 — 필요한 기능들

Protégé에 있는데 OntologyHub에 없는 것 + 도입 우선순위

CRITICAL

논리적 일관성 검증

현재 모순 노드를 잡지 못함. "채식주의자" + "삼겹살 좋아함"이 공존 가능.

방안: Gemini 모순 탐지 프롬프트 + 경량 규칙 엔진
CRITICAL

표준 포맷 내보내기

OWL 2 / JSON-LD 지원 시 전세계 온톨로지 도구와 호환. MCP 서버에 필수.

방안: ALIVE Schema → OWL 2 매핑 + rdflib 변환기
HIGH

08. 소스코드 적용 분석

126개 리포지토리 중 OntologyHub에 활용 가능한 부분 점검

핵심 판단: 소스코드를 직접 가져다 쓰기는 어렵다

Protégé는 100% Java로 작성되었고, 우리 백엔드는 Python (FastAPI)입니다. Java 코드를 직접 포팅하는 것은 비효율적입니다.

하지만, Protégé의 아키텍처 패턴, 데이터 모델, 알고리즘 설계는 그대로 참고할 수 있고, 같은 기능을 하는 Python 라이브러리(owlready2, rdflib)를 사용해 구현할 수 있습니다.

특히 Neo4j의 Neosemantics(n10s) 플러그인은 우리 Neo4j Aura와 직접 연동되어, RDF/OWL↔Neo4j 변환을 바로 사용할 수 있습니다.

09. 모듈별 적용 방안

Protégé 소스코드의 어떤 부분을 우리 프로젝트에 어떻게 활용하는지

✅ 바로 적용 가능

OWL 2 / JSON-LD 내보내기

protege → model/io/ | Python: rdflib + owlready2

Protégé의 IO 모듈은 6개 포맷 내보내기를 지원. Python의 rdflib가 동일한 기능을 제공합니다.

# 우리 백엔드에 추가할 코드 (Python) from rdflib import Graph, Namespace, URIRef, Literal from rdflib.namespace import RDF, RDFS, OWL def convert_neo4j_to_owl(neo4j_nodes, neo4j_edges):   g = Graph()   ALIVE = Namespace("https://ontologyhub.ai/schema#")   g.bind("alive", ALIVE)   for node in neo4j_nodes:     g.add((ALIVE[node.id], RDF.type, OWL.NamedIndividual))     g.add((ALIVE[node.id], RDFS.label, Literal(node.name)))   for edge in neo4j_edges:     g.add((ALIVE[edge.source], ALIVE[edge.type], ALIVE[edge.target]))   return g.serialize(format="json-ld")
🔄 변환 적용

추론 엔진 (Reasoner) 연동

protege → model/inference/ | Python: owlready2 (HermiT 내장)

owlready2는 HermiT와 Pellet 추론기를 내장하고 있습니다.

from owlready2 import * onto = get_ontology("file://alive_schema.owl").load() with onto:   sync_reasoner_hermit(infer_property_values=True) inconsistent = list(onto.inconsistent_classes()) if inconsistent:   print(f"모순 발견: {inconsistent}")
